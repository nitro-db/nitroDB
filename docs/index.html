NEDB

A versioned, time-traveling embedded database. Replay-protected, durable, relational, searchable — with an append-only log you can prove, and a server daemon when you need one.

View on GitHub PyPI
$ pip install nedb-engine
quickstart.py
from nedb import NEDB

db = NEDB("./data")            # durable: append-only log on disk
db.create_index("users", "status", "eq")
db.put("users", "alice", {"name": "Alice", "status": "active"})

db.query('FROM users WHERE status = "active" ORDER BY name')
db.get("users", "alice", as_of=0)   # time-travel
assert db.verify()                  # hash chain intact
Why NEDB

Redis-class speed, with the things real systems need

Redis is fast because it's in-memory and simple — but relations are hand-rolled, history is gone the moment you overwrite, and every call pays a hop. NEDB keeps the speed and builds idempotency, replay protection, MVCC, time-travel, and durability into one hash-chained, append-only log.

⛓️

Replay-protected log

Every write carries a monotonic per-client nonce + optional idempotency key. Retries are no-ops; tampering is detectable via verify().

🕰️

Time-travel

Read the database exactly as it was at any past sequence — AS OF seq. MVCC snapshots and deterministic replay fall out of the same log.

💾

Durable, Redis-style

Point a database at a path and every op is appended to disk and fsync'd; it reloads by replaying the log. verify() and AS OF survive restarts.

🔗

First-class relations

Adjacency-list graph edges with O(1) traversal — and the graph time-travels too.

🔎

Filter · sort · search

Equality, ordered, and full-text inverted indexes, maintained incrementally. One small query language: NQL.

📦

git-style files

Content-defined chunking + dedup + compression tiers. Every file version gets a Merkle root you can anchor on-chain.

Release highlights

The 0.3 line

v0.3.1

Universal install

Pure-Python wheel + sdist — pip install nedb-engine works on every platform and Python, and always ships the nedbd command.

v0.3.0

nedbd — the server daemon

Run NEDB as a long-lived HTTP/JSON server. Each database is durable and held open in memory; connect over a URL like Redis or Postgres.

v0.2.0

Durable persistence

Append-only log (AOF) on disk + load-by-replay. The tamper-evident chain — and time-travel — now survive a restart.

Run it as a server

nedbd

pip install nedb-engine ships a daemon. Run the engine as its own process and connect clients over HTTP — the way you'd run Redis or Postgres. Each named database is durable and held open in memory.

shell
$ nedbd                 # http://127.0.0.1:7070

$ curl -X POST localhost:7070/v1/databases \
    -d '{"name":"shop","init":{"seed":{"users":[{"id":"u1","name":"Ada"}]}}}'

$ curl -X POST localhost:7070/v1/databases/shop/query \
    -d '{"nql":"FROM users WHERE name = \"Ada\""}'
# env: NEDBD_HOST · NEDBD_PORT · NEDBD_DATA · NEDBD_TOKEN
Install

Two lines to a database

pip install nedb-engine

NEDB() is in-memory; NEDB("./path") is durable. nedbd runs it as a server. The pure-Python reference is the always-works core; a native Rust core is additive (roadmap). Read the format spec.
